# x402 — pay protocol-gated endpoints from a Crossmint wallet

> **This file overrides the live `docs.crossmint.com/agents/x402` page.** The live doc shows `wrapFetchWithPayment` from `@x402/core/client`, but that helper was **removed in `@x402/core ≥ 2.11.0`**. Use the manual flow below — it's what works against current packages and against both x402 v1 and v2 endpoints.

## Mandatory order of operations

1. **Probe the endpoint first.** Send a no-payment request and read the 402 body. **Never skip this.** You can't know the version, network, amount, or recipient until you do.
2. **Confirm with the user** the version, amount, and target before signing. Even $0.01 is real money.
3. **Sign + retry** with the right header for the protocol version.
4. **Decode the receipt** from `x-payment-response` to confirm settlement.

## Prerequisites

- A Crossmint EVM wallet on the **same chain the endpoint accepts** — most live endpoints want **base-sepolia** (staging) or **base** (production) USDC. Verify after probing.
- USDC balance on that chain. **USDXM does not work** — no live x402 endpoint accepts it. For staging, fund via [faucet.circle.com](https://faucet.circle.com/) (paste address, pick Base Sepolia, no auth). See `references/funding-staging-wallets.md`.
- The wallet has an active server signer: `await wallet.useSigner({ type: "server", secret: process.env.CROSSMINT_SIGNER_SECRET })`.

## Step 1 — probe

```bash
curl -si https://api.example.com/protected
```

The response is `HTTP 402` with a JSON body like:

```json
{
  "x402Version": 1,
  "accepts": [
    {
      "scheme": "exact",
      "network": "base-sepolia",
      "asset":   "0x036CbD53842c5426634e7929541eC2318f3dCF7e",   // USDC contract
      "payTo":   "0x6813749E1...76cdD0",
      "maxAmountRequired": "5000",                                 // raw USDC (6 decimals → $0.005)
      "resource": "https://api.example.com/protected"
    }
  ]
}
```

Read off:
- `x402Version` → `1` or `2`. Determines which scheme + which header name to use (table below).
- `accepts[0].network` → `base-sepolia`/`base` for v1, `eip155:84532`/`eip155:8453` for v2.
- `accepts[0].maxAmountRequired` → raw USDC. Divide by 1,000,000 for human dollars.

## Step 2 — confirm

Tell the user the exact action:
> Endpoint requires up to **<amount> USDC** on **<network>** to **<endpoint URL>**. Pay? (y/N)

Wait for explicit yes before signing.

## Step 3 — sign + retry

### Install
```bash
npm i @crossmint/wallets-sdk @x402/core @x402/evm viem
```

### Working snippet (v1 + v2 supported, register both)

```typescript
import { createCrossmint, CrossmintWallets, EVMWallet } from "@crossmint/wallets-sdk";
import { x402Client } from "@x402/core/client";
import { encodePaymentSignatureHeader } from "@x402/core/http";
import { ExactEvmSchemeV1 } from "@x402/evm/exact/v1/client";
import { ExactEvmScheme }   from "@x402/evm/exact/client";

const crossmint = createCrossmint({ apiKey: process.env.CROSSMINT_API_KEY });
const wallets = CrossmintWallets.from(crossmint);

// 1. Get wallet + activate signer
const wallet = await wallets.getWallet(
  "evm:alias:claude-agent-wallet",                      // or the 0x address
  { chain: "base-sepolia" }                              // must match endpoint's network
);
await wallet.useSigner({ type: "server", secret: process.env.CROSSMINT_SIGNER_SECRET });
const evmWallet = EVMWallet.from(wallet);

// 2. Adapt to x402's signer interface
const x402Signer = {
  address: evmWallet.address,
  async signTypedData(typedData) {
    const { signature } = await evmWallet.signTypedData({
      ...typedData,
      chain: "base-sepolia",                              // keep in sync with wallet chain
    });
    return signature;
  },
};

// 3. Register BOTH versions — v1 uses network names, v2 uses CAIP-2.
//    You don't know which until you probe; cheaper to register both up front.
const client = new x402Client();
client.register("eip155:*", new ExactEvmScheme(x402Signer));               // v2 (any EVM)
client.registerV1("base-sepolia", new ExactEvmSchemeV1(x402Signer));       // v1 (base-sepolia)
// For mainnet add:
//   client.registerV1("base", new ExactEvmSchemeV1(x402Signer));

const TARGET = "https://api.example.com/protected";

// 4. Probe
const probe = await fetch(TARGET, { method: "GET" });
if (probe.status !== 402) {
  // Not an x402 endpoint, or already returned content. Print and stop.
  console.log("status:", probe.status, "body:", await probe.text());
  process.exit(0);
}

// 5. Parse requirements (this is the structured 402 body)
const paymentRequired = await probe.json();
console.log("version:", paymentRequired.x402Version,
            "max:", paymentRequired.accepts[0].maxAmountRequired,
            "network:", paymentRequired.accepts[0].network);

// 6. Sign payment payload (handles HKDF derivation + EIP-3009 transferWithAuthorization)
const paymentPayload = await client.createPaymentPayload(paymentRequired);

// 7. Encode the header — name depends on version
const headerValue = encodePaymentSignatureHeader(paymentPayload);
const headerName  = paymentPayload.x402Version === 1 ? "X-PAYMENT" : "PAYMENT-SIGNATURE";

// 8. Retry with payment header (use the SAME method as the original endpoint)
const paid = await fetch(TARGET, {
  method: "GET",
  headers: { [headerName]: headerValue },
});

console.log("paid status:", paid.status);
console.log("paid body:",   await paid.text());

// 9. Decode the settlement receipt from response header
const receiptB64 = paid.headers.get("x-payment-response");
if (receiptB64) {
  const receipt = JSON.parse(Buffer.from(receiptB64, "base64").toString());
  console.log("receipt:", receipt);
  // { success: true, transaction: "0x...", network: "base-sepolia", payer: "0x..." }
}
```

## Version → header table

| `x402Version` | Network field example | Scheme class | Register call | Request header |
|---|---|---|---|---|
| **1** | `"base-sepolia"`, `"base"` | `ExactEvmSchemeV1` | `client.registerV1(name, scheme)` | `X-PAYMENT` |
| **2** | `"eip155:84532"`, `"eip155:8453"` | `ExactEvmScheme` | `client.register("eip155:*", scheme)` | `PAYMENT-SIGNATURE` |

Both are EIP-3009 `transferWithAuthorization` underneath; the wire format differs.

## Common gotchas (each one cost a real session)

- **`wrapFetchWithPayment` does not exist in `@x402/core ≥ 2.11.0`.** Use the manual flow above. The Crossmint live docs and earlier x402 examples are stale on this.
- **Network mismatch is silent.** If your wallet is on `base-sepolia` and the endpoint accepts only `base` mainnet, the SDK will sign anyway but the facilitator will reject. Always cross-check `wallet chain == accepts[0].network`.
- **v1 vs v2 confusion.** Register both schemes up front. If you only register v2 and probe a v1 endpoint, you'll see `"no payment scheme accepted for network: base-sepolia"`.
- **Wrong header name.** v1 uses `X-PAYMENT`, v2 uses `PAYMENT-SIGNATURE`. Disambiguate from `paymentPayload.x402Version`.
- **Payment payload expires.** EIP-3009 authorizations have a `validBefore` (~15 min). If you sign once, hold the header, then retry minutes later → 500 from facilitator. Re-sign for each retry.
- **HTTP 405 after a successful payment is an endpoint bug, not a payment bug.** Check `x-payment-response` — if `success: true`, the USDC moved. The endpoint just failed to serve content. Tell the user the payment landed and show the tx hash.
- **Same wallet on both networks.** Server-signer wallets at the same `(secret, project, alias)` give the same address on `base-sepolia` and `base`, but USDC balances are separate (different tokens on different chains).

## When the user says "use x402 to pay this URL"

The agent should:
1. **Always** start with `curl -si <URL>`. Do not skip.
2. Read off version, network, amount, payee from the 402 body.
3. Cross-check the wallet's chain matches `accepts[0].network`.
4. Cross-check balance ≥ `maxAmountRequired / 1_000_000`. If short, point at `funding-staging-wallets.md`.
5. Show the user: "Probed. Endpoint wants up to **$X.XX USDC** on **<network>** (x402 v<N>). Pay?"
6. On confirm, run the snippet above with `TARGET` set to the URL and the chain matching `accepts[0].network`.
7. Print the receipt from `x-payment-response`.
